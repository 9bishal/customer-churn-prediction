## 1. What is Customer Churn?

**Churn** occurs when a customer cancels their subscription or stops doing business with a company. 
* **The Goal**: Predict which customers are at risk of leaving *before* they cancel, giving retention teams a chance to offer discounts or incentives to keep them.
* **Why it matters**: Keeping existing customers is significantly cheaper (5x to 7x) than acquiring new ones.

## 2. Directory & File Breakdown

Here is a map of the project files and what each one does:

### 📂 `data/`
* `data/raw/telco_churn.csv`: The original, unaltered dataset containing information about 7,043 historical customers (demographics, services used, contract length, charges, and whether they eventually churned).
* `data/processed/churn_clean.csv`: The cleaned and prepared dataset. This file is generated by running `src/data_preprocessing.py` and is the direct input used to train the machine learning models.

### 📂 `src/` (Source Code)
This folder contains the core Python scripts that handle the data pipeline and modeling:
* [`data_preprocessing.py`](file:///Users/bishalkumarshah/Downloads/ML%20Projects/customer-churn-prediction/src/data_preprocessing.py): 
  * **Role**: Cleans raw data (fills blank total charges with 0, removes unused columns like IDs).
  * **Feature Engineering**: Creates new features like `tenure_group` (grouping customers by how many months they've been with the company) and `avg_monthly_spend`.
  * **Categorical Encoding**: Converts text categories (e.g., "Month-to-month", "DSL") into numerical columns ($0$ or $1$) so the model can process them.
* [`train_model.py`](file:///Users/bishalkumarshah/Downloads/ML%20Projects/customer-churn-prediction/src/train_model.py): 
  * **Role**: Trains candidate models (Logistic Regression and Random Forest), compares their performance, and saves the best one.
  * **Evaluation**: Measures precision, recall, F1-score, and ROC-AUC. It selects the model with the best **Recall** (catching the most churners).
* [`predict.py`](file:///Users/bishalkumarshah/Downloads/ML%20Projects/customer-churn-prediction/src/predict.py): 
  * **Role**: The bridge between the ML model and the user interface. It takes a raw, human-readable input (e.g., from the dashboard form), runs it through the preprocessing pipeline, and outputs a churn probability (e.g., 85% risk).

### 📂 `models/`
* `models/churn_model.pkl`: The saved ("serialized") version of our trained pipeline. It contains both the scaler (for normalizing numeric values) and the trained model. `predict.py` loads this file to make real-time predictions.

### 📂 `app/` (User Interface)
* [`streamlit_app.py`](file:///Users/bishalkumarshah/Downloads/ML%20Projects/customer-churn-prediction/app/streamlit_app.py): 
  * **Role**: An interactive web interface built with Streamlit. Users can input customer details using sliders and dropdowns and immediately see the predicted churn risk and main factors driving the decision.

### 📂 `notebooks/` (Exploration)
* `notebooks/eda.ipynb`: A Jupyter Notebook used for **Exploratory Data Analysis (EDA)**. This is where the developer analyzed data distributions, made plots, and discovered insights (e.g., month-to-month contract holders have the highest churn rate).

### ⚙️ Configuration & Metadata
* [`requirements.txt`](file:///Users/bishalkumarshah/Downloads/ML%20Projects/customer-churn-prediction/requirements.txt): Lists all the third-party Python packages and exact versions (e.g., `pandas`, `scikit-learn`, `streamlit`) needed to run this project.

## 3. How the Data Flows

```
[Raw Data] -> [data_preprocessing.py] -> [Processed Data]
                                                |
                                                v
[Trained Model (.pkl)] <- [train_model.py] <----+
        |
        v
[predict.py] <--- [streamlit_app.py] (Dashboard User Input)
        |
        v
   [Prediction] (Churn Probability & Risk Tier)
```

1. **Preprocessing**: Raw CSV is cleaned and engineered into a processed CSV.
2. **Training**: `train_model.py` splits the processed data into training (to teach the model) and test sets (to evaluate the model). The final best model is saved as `churn_model.pkl`.
3. **Serving**: The Streamlit app takes user input, passes it to `predict.py`, which loads `churn_model.pkl` to calculate the churn probability and outputs the prediction to the screen.

## 4. Key ML Concepts to Know

* **Feature Engineering**: Creating new inputs for the model from existing data. For example, bucketizing `tenure` (months) into `tenure_group` makes it easier for the model to see the difference between brand-new customers and long-term loyal ones.
* **Pipeline**: A sequence of steps (scaling numbers + running the model) packaged into one object. This ensures the data is processed in the *exact* same way during training and when making predictions in Streamlit, avoiding bugs.
* **Recall vs. Precision**:
  * **Recall** (Sensitivity) measures what percentage of actual churners the model caught. 
  * **Why Recall is preferred here**: If we miss a customer who is about to leave (False Negative), we lose their revenue forever. If we flag a loyal customer as about to leave (False Positive), we might just send them an unnecessary promotional discount—which is much less costly.
* **Pickle (`.pkl`)**: A way of saving Python objects directly to disk. In ML, we "pickle" our trained models so we can reuse them later without retraining them from scratch.
